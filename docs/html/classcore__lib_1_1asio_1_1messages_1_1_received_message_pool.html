Detailed Description

template<typename Header>
class core_lib::asio::messages::ReceivedMessagePool< Header >

Reusable message pool class.

The MessageHandler class now uses this specialised to our MessageHeader. This pool is smart, if it cannot acquire a message from the pool, either because there are no free messages or the required message size is too bg then it falls back to dynamic allocation.

The really nice thing is the pool messages release themselves pack to the pool once the last instance of the shared_ptr to the message goes out of scope. Dynamically allocated "fallback" messages self-destruct as normal through the shared_ptr's default destruction logic.

Header must be default constructable.
