Parameters
[in]ioService- External boost IO service to manage ASIO.
[in]listenPort- Our listen port for all detected networks.
[in]messageDispatcher- Callback to use to dispatch received messages.
[in]receiveOptions- Socket receive option to control the use of broadcasts/unicast.
[in]receiveBufferSize- Socket receive option to control receive buffer size.

Typically use this constructor when managing a bool of threads using an instance of core_lib::asio::IoServiceThreadGroup in your application to manage a pool of std::threads. This means you can use a single thread pool and all ASIO operations will be exectued using this thread pool managed by a single IO service. This is the recommended constructor.
